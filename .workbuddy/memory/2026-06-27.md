# 2026-06-27 AuditAI 工作日志

## TicketNavGrid 虚拟节点不显示 BUG 修复

**文件**：`AuditAI/Auditai.UI.Platform/TicketNavGrid.cs`

**现象**：表单导航树的"单位名称"列预设虚拟节点（蓝色 ComboList 节点）在内存中已正确创建（`Created new virtual node ×2`，`DisplayStringRealValue` 正确设为单位名称），但界面不显示。

**根因**：`Populate()` 方法中 `i` 和 `level` 变量被错误地声明在 `foreach` 循环内部，导致每个根子节点处理时 `i` 都从 `fixedRowsCount` 重置，后续子节点覆盖前面已写入的 Grid 行。最终只有最后一个根子节点可见，前面的实际记录节点和虚拟节点全部被覆盖。

**修复**：将 `var i = fixedRowsCount; var level = 0;` 移到 `foreach` 循环之外，让 `i` 通过 `ref` 在所有 `AddNode` 调用之间累加。

**对照原版验证**：通过 mono-cecil-method-restore skill 提取 `E:/lq/LeqiAudit/LeqiAudit.exe` 中 `Leqisoft.UI.Platform.TicketNavGrid` 的 IL：
- 原版 `Populate` IL_0094-009F：`i` 和 `level` 在 foreach 之前只初始化一次，通过 `DisplayClass122_1` 的 ref 字段在所有 AddNode 调用间共享累加。
- 原版 `AddNode` IL_008D-0098：`i = i + 1` 通过 `stfld` 写回 DisplayClass（ref 语义）。

**对比发现的其他差异（暂未修复）**：
1. 原版 `_grid_BodyOwnerDrawCell` 只用 `navNode.Text` 作为显示文本，当前代码优先用 `DisplayStringRealValue`（过度设计，但原版 AddLeafNode 中 Item1=Item2=Text，所以两者实际等价）。
2. 原版 `_grid_BodyOwnerDrawCell` 有完整的 Image 设置逻辑（叶子节点用 Ticket16/VirtualTicket16，父节点按 Collapsed 状态用 TicketNavTreeListCollapsed/Expanded 等），当前代码完全缺失，虚拟节点图标不会显示。
3. 原版 `Populate` 没有 `Grid.Cols.Count = Nav.Count` 和 `Grid.Cols[i].Name/Width` 的设置（当前代码额外加了，不影响功能）。

## 数据库链路排查（第二轮 - 用户反馈 Populate 修改无效）

用户反馈：修改 TicketNavGrid.Populate 的 i/level 作用域后虚拟节点仍不显示，要求从数据库找"应收账款替代测试表"追踪代码读取链路。

### 数据库定位结果（项目库 92410a46-...db）

1. **目标表**：TreeNode Id=11287174055345「应收账款替代测试表」→ Table.Id 同值
   - Ticket JSON: `Navs=[{"Columns":[11287174055451]}]`, `IsAllowShowVirtualNode=true`, `DataRowStart=2, DataRowCount=6`, Rows=11, Cells=121
2. **Nav 列**：Column 表 Id=11287174055451「单位名称」, Index=4, StyleId=11476152614921
3. **ComboList 公式**：CellStyle 表 Id=11476152614921 的 Format 字段 `{"T":0,"D":0,"L":"TableList([2:10337986289169:10337986289222]<>\"\",[2:10337986289169:10337986289222],[2:10337986289169:10660108840281],[2:10337986289169:10337986289224],[2:10337986289169:10337986289226],[2:10337986289169:10337986289228],[2:10337986289169:10337986289230])","IgnoreComboList":true}`
   - ComboList 公式在 Format 的 `L` 字段（DataFormat.ComboList 属性，JsonProperty "L"）
   - **IgnoreComboList=true**（但不影响 FillVirtualNode，因为 GetContainsComboListColumn 只检查 HasComboList=L非空）
4. **ComboList 引用的源表**：Table Id=10337986289169「应收账款明细表替代测试表（前半）」
   - Ticket JSON **完全空**（Columns=[], Rows=[], Cells=[], DataRowCount=0）
   - 但 Row 表有 9 行，Cell 表有 54 个单元格（9×6）
   - Column 10337986289222（单位名称）的 9 个单元格值：3 个非空（石家庄高新区/藁城区/长安区优宝贝幼儿园）+ 空行 + 合计/审计说明/审计结论

### 代码读取链路

1. `TableInputEditor2.cs:6550` `ticketNavGrid.Nav = ticketNav2.Columns.Select(id => Table.Columns.GetById(id)).ToList()` — 从 Ticket.Navs 构建 Nav 列表
2. `Table.cs:464-516` 加载 CellStyle（含 ComboList 的 L 字段）→ 关联到 Column.Style
3. `Table.cs:518-566` 从 Row/Cell 表加载行和单元格到 Table.Rows/Cells
4. `TicketNavGrid.MakeTree` → `FillVirtualNode` → `GetComboListValue(column)` 求值 ComboList 公式
5. `GetComboListValue` 用 `FormulaEvaluator` 求值 `TableList([2:10337986289169:...])`，通过 `FormulaReferenceModelResolver` 解析表引用

### 最可能的根因方向

- **源表 Ticket JSON 空**：源表 10337986289169 的 Table.Ticket JSON 是空的，但 Row/Cell 表有数据。Table 加载时从 Row/Cell 表独立加载（Table.cs:518-566），Table.Rows 应有 9 行。
- **ComboList 求值断裂**：GetComboListValue 求值时，FormulaReferenceModelResolver 可能找不到/未加载源表 10337986289169，或求值异常被 catch 静默吞掉（返回 null），导致虚拟节点不生成。
- **关键验证点**：需确认 GetComboListValue 实际返回值（Debug 日志 `result count=`），以及源表是否在求值时已加载到 Project。
